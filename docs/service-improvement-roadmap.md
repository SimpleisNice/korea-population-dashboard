## 1. 서비스 현황 요약

### 1.1 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 대한민국 거주 인구 통계 대시보드 |
| 타깃 사용자 | 부동산 구매·투자 검토 중인 개인, 이주 지역 리서치 중인 사용자 |
| 데이터 출처 | 행정안전부(MOIS) 주민등록 인구 통계 |
| 데이터 범위 | 269개 시군구 × 40개월 (2023.01 ~ 2026.04) |
| 갱신 주기 | 월 1회 수동 업데이트 |

### 1.2 현재 제공 기능

| 페이지 | 경로 | 핵심 기능 |
|--------|------|-----------|
| 홈 | `/` | 지역 퍼지 검색, 인기 지역 6개(하드코딩), 최근 본 지역(localStorage) |
| 지역 상세 | `/[sido]/[sigungu]` | 최신 인구·세대·전월 증감 카드, 12개월 추이 차트, 시점 비교 |
| 상세 탭 | `/[sido]/[sigungu]/detail` | 인구추이 / 세대 / 연령 분포 / 월별 증감 (4탭) |
| 비교 | `/compare` | 두 지역 A·B 주요 지표 비교 테이블 + 추이 오버레이 차트 |

### 1.3 데이터로 계산 가능한 지표

현재 코드에 존재하는 원시 데이터 기준, 추가 구현 없이 바로 산출 가능한 파생 지표:

- 전년 동월 증감 (YoY)
- 증감률 (%)
- 성비 (남성 / 여성 × 100)
- 고령화 지수 (65세 이상 / 14세 이하 × 100)
- 생산가능인구 비율 (15~64세 / 총인구 × 100)
- 1인 세대 비율 (세대 수 대비 추정 불가 — 원시 데이터 미포함)

## 2. 사용자 시나리오 & 페인 포인트

### 핵심 사용자 시나리오

> **"경기도 판교 근처로 이사를 고려 중이다. 인구가 꾸준히 늘고 있는지, 젊은 층 비중이 높은지, 주변 다른 지역과 비교했을 때 어떤지 확인하고 싶다."**

| 단계 | 현재 사용자 행동 | 페인 포인트 |
|------|-----------------|-------------|
| 지역 탐색 | 홈에서 "판교"로 검색 → 분당구 이동 | 인기 지역 추천이 부동산 관심도 없는 강남·송파 고정 — 개인화 없음 |
| 인구 동향 확인 | 지역 상세에서 12개월 차트 확인 | 전년 대비 얼마나 늘었는지 YoY 수치가 없음 |
| 장기 추세 판단 | 12개월 차트만 볼 수 있음 | 40개월 데이터가 있음에도 최근 1년 외엔 볼 수 없음 |
| 연령 구조 판단 | 연령 탭에서 현재 분포 확인 | 이 지역이 고령화되는 추세인지 시간 변화 확인 불가 |
| 주변 지역 비교 | 비교 페이지에서 A·B 설정 | 고령화지수·성비 등 심층 지표 없음, 2개 지역만 비교 가능 |
| 상대적 위치 파악 | 확인 수단 없음 | 분당구가 경기도에서 인구 몇 위인지 알 수 없음 |
| 데이터 저장·공유 | 확인 수단 없음 | 스크린샷 외 공유 방법 없음, 재방문 시 처음부터 설정 반복 |

## 3. 기능 개선 사항

### I1. 전년 동월(YoY) 비교 추가

**현재 상태**  
지역 상세 페이지의 StatCard가 전월 대비(MoM) 증감만 표시한다.

**문제**  
부동산 구매자는 계절성을 제거한 1년 단위 변화가 이사 결정에 더 유효한 시그널이다. "전월보다 200명 감소"는 맥락 없이 불안하게 보이지만, "전년 동월 대비 1,200명 증가"면 긍정 신호임을 바로 알 수 있다.

**개선 방향**
- StatCard에 전월비(MoM)와 전년 동월비(YoY) 두 줄 동시 표시
- YoY 계산: `data[ym] - data[ym - 12개월]`  
- 데이터 없을 경우(초기 12개월 이전) YoY 항목 숨김 처리

**영향 범위**
- `src/components/region/StatCard.tsx`
- `src/lib/data.ts` — `getRegionDetail()` 반환값에 `yoyChange` 필드 추가

**임팩트** High | **개발 규모** S (1~2일)

---

### I2. 인기 지역 동적 산출

**현재 상태**  
`src/lib/data.ts`의 `getPopularRegions()`가 강남구·송파구·분당구·마포구·해운대구·화성시 6개 코드를 하드코딩으로 반환한다.

**문제**  
실제 인구 증가 추세와 무관한 지역이 추천되어 서비스 신뢰도 저하. 데이터 기반 서비스라는 포지셔닝과 불일치.

**개선 방향**  
최근 12개월 인구 증가율 상위 6개 지역을 동적 산출로 교체.  
단, 전국 1~2위 광역시(서울·경기)가 독점하지 않도록 시도별 1개 제한 옵션 검토.

**영향 범위**
- `src/lib/data.ts` — `getPopularRegions()` 로직 교체
- 빌드 시간에 계산하여 캐싱 (런타임 성능 영향 없음)

**임팩트** Medium | **개발 규모** S (1일)

---

### I3. 추세 차트 기간 선택

**현재 상태**  
지역 상세 및 비교 페이지의 TrendChart가 항상 최근 12개월만 표시한다.

**문제**  
40개월 데이터가 있음에도 장기 트렌드(2~3년 추세선, 하락 전환점) 확인이 불가능하다. 부동산 사이클을 판단하려는 사용자에게 정보 부족.

**개선 방향**
- 차트 상단에 "6개월 / 12개월 / 전체" 세그먼트 컨트롤 추가
- 선택 값을 URL 파라미터(`range=6|12|all`)로 관리
- 전체 기간 선택 시 X축 레이블 간격 자동 조정 (3개월마다 표시)

**영향 범위**
- `src/components/region/TrendChart.tsx`
- `src/components/compare/CompareClient.tsx`
- URL 파라미터 추가

**임팩트** Medium | **개발 규모** M (2~3일)

### I4. 연령 분포 시간 변화 비교

**현재 상태**  
`/detail` 연령 탭이 선택된 단일 월의 스냅샷만 표시한다.

**문제**  
"이 지역이 고령화되고 있는가?" 라는 핵심 질문에 답할 수 없다. 40개월치 연령 데이터가 모든 지역에 존재함에도 활용되지 않는다.

**개선 방향**  
연령 탭 내에 "비교 시점" 월 선택기 추가.  
선택 두 시점의 연령 분포를 같은 차트에 오버레이 (반투명 처리로 구분).  
기본값: 현재 월 vs 24개월 전.

**영향 범위**
- `src/components/detail/AgeChart.tsx` — 이중 데이터셋 지원
- `src/app/[sido]/[sigungu]/detail/page.tsx` — 두 번째 월 데이터 fetch 추가

**임팩트** High | **개발 규모** M (3일)

### I5. 비교 지표 확장

**현재 상태**  
비교 페이지의 지표 테이블이 인구·세대·세대당 인구·남성·여성 5개로 고정되어 있다.

**문제**  
성비, 고령화 지수, YoY 증감률 등 부동산 투자자가 실제로 보고 싶은 지표가 없다.

**개선 방향**
- 비교 테이블 항목 확장 (최대 8개):
  - 기존 5개 유지
  - 신규 추가: 성비, 전년 동월 증감률(YoY%), 고령화 지수
- 각 지표에 툴팁으로 계산 공식 설명 추가

**영향 범위**
- `src/components/compare/CompareClient.tsx`
- `src/lib/data.ts` — `getMonthStats()` 반환값 확장

**임팩트** Low-Medium | **개발 규모** S (1~2일)

### I6. 지역 내 순위 컨텍스트 배지

**현재 상태**  
지역 상세 페이지에 절댓값 수치만 있고 상대적 위치 정보가 없다.

**문제**  
"강남구 인구 558,000명"이라는 숫자 단독으로는 큰지 작은지 알 수 없다. 맥락 없는 숫자는 판단 기준을 제공하지 못한다.

**개선 방향**
- 페이지 상단 또는 총인구 카드 하단에 배지 1줄 추가  
  예: `서울특별시 내 인구 3위 · 전국 15위`
- 빌드 타임에 전 지역 순위 사전 계산 (런타임 비용 없음)

**영향 범위**
- `src/lib/data.ts` — 순위 계산 유틸 추가
- `src/app/[sido]/[sigungu]/page.tsx` — 배지 컴포넌트 추가

**임팩트** Medium | **개발 규모** S (1일)

## 4. 신규 기능 제안

### N1. 지역 순위 페이지 `/ranking`

**가치 제안**  
"어느 지역이 요즘 인구가 가장 빠르게 늘고 있나?" 라는 탐색 니즈를 충족. 직접 지역명을 모르는 사용자가 서비스에 진입하는 신규 진입점.

**기능 명세**

| 항목 | 내용 |
|------|------|
| 정렬 기준 | 총 인구 / 전월 증감 / 전년 동월 증감률 / 세대 수 |
| 필터 | 전체 / 시도 선택 (드롭다운) |
| 표시 항목 | 순위, 지역명, 선택 지표 값, 변화 방향 아이콘 |
| 페이징 | 무한 스크롤 또는 20개씩 페이지네이션 |
| 지역 클릭 | 해당 지역 상세 페이지로 이동 |

**필요 데이터**  
기존 데이터 100% 활용. 신규 데이터 불필요.

**개발 규모** M (3~4일) | **임팩트** High

---

### N2. 인구 급증·급감 탐색 `/trending`

**가치 제안**  
"지금 뜨는 지역"과 "빠지는 지역"을 한눈에 보여줌. 부동산 투자자와 이주 검토자 모두에게 액션어블한 인사이트 제공. SEO 및 재방문율 개선 기대.

**기능 명세**

| 항목 | 내용 |
|------|------|
| 섹션 | 급증 TOP 10 / 급감 TOP 10 |
| 기간 선택 | 3개월 / 6개월 / 12개월 |
| 표시 항목 | 지역명, 변화 인원, 변화율(%), 방향 아이콘 |
| 업데이트 | 데이터 업데이트 시 자동 반영 |
| 레이아웃 | 카드 리스트 (모바일 세로 스크롤) |

**필요 데이터**  
기존 데이터 100% 활용.

**개발 규모** M (3일) | **임팩트** High

---

### N3. 전국 총괄 현황 — 홈 상단 배너

**가치 제안**  
서비스 진입 시 "대한민국 전체 인구 현황"을 즉시 파악. 서비스 신뢰도·전문성 강화. 사용자 체류 시간 증가.

**기능 명세**

| 항목 | 내용 |
|------|------|
| 표시 위치 | 홈 검색창 위 또는 아래 |
| 표시 항목 | 전국 총 인구, 전월 대비 증감, 기준 연월 |
| 디자인 | 카드 1개, 간결한 수치 표시 |
| 링크 | 탭/버튼으로 `/ranking` 또는 `/trending` 진입 유도 |

**필요 데이터**  
전체 지역 인구 합산. 기존 데이터 활용.

**개발 규모** S (1일) | **임팩트** Medium

### N4. 즐겨찾기 (관심 지역)

**가치 제안**  
재방문 사용자가 매번 검색 없이 관심 지역에 바로 접근. 부동산 검토는 수주~수개월에 걸쳐 반복 방문하는 특성상 전환율 향상 직결.

**기능 명세**

| 항목 | 내용 |
|------|------|
| 저장 위치 | localStorage (서버 계정 불필요) |
| 최대 개수 | 10개 |
| 진입 | 지역 상세 페이지 상단 북마크 아이콘 |
| 표시 | 홈 "관심 지역" 섹션 (최근 본 지역 바로 위) |
| 순서 | 추가된 순서, 드래그 재정렬은 Phase 2 고려 |

**필요 데이터**  
없음. 순수 클라이언트 기능.

**개발 규모** S (1~2일) | **임팩트** Medium

### N5. 연령 구조 심층 지표

**가치 제안**  
연령 분포 원시 데이터를 부동산 투자 판단에 유용한 파생 지표로 변환. "이 지역은 어린 가족 중심인가, 고령화 진행 중인가"를 수치로 제공.

**기능 명세**

| 지표 | 계산식 | 해석 |
|------|--------|------|
| 고령화 지수 | 65세 이상 / 14세 이하 × 100 | 100 초과 = 고령화 진행 |
| 생산가능인구 비율 | 15~64세 / 총인구 × 100 | 높을수록 경제 활동 활발 |
| 성비 | 남성 / 여성 × 100 | 100 = 균형 |
| 중위 연령대 (추정) | 가중 평균 연령대 | 지역 생애 주기 파악 |

- 표시 위치: `/detail` 연령 탭 상단 요약 카드 4개
- 툴팁으로 계산식 설명

**필요 데이터**  
기존 연령 분포 데이터 활용.

**개발 규모** M (2일) | **임팩트** Medium

---

### N6. 공유 & 내보내기

**가치 제안**  
부동산 카페, 가족 단톡방 등에서 "이 지역 인구 데이터 봐봐"로 바로 공유 가능. 데이터 기반 논의 문화 형성. 바이럴 유입 경로 생성.

**기능 명세**

| 기능 | 상세 |
|------|------|
| URL 복사 | 현재 뷰(지역 + 월 + 탭)의 URL을 클립보드에 복사 |
| 이미지 저장 | 현재 차트 영역을 PNG로 저장 (html2canvas 활용) |
| CSV 다운로드 | 해당 지역 전체 월별 데이터 CSV 다운로드 |

- 진입: 지역 상세 페이지 우상단 공유 아이콘 → 바텀시트
- URL 공유는 이미 nuqs로 URL 상태 관리가 되어 있어 기술적 준비 완료

**필요 데이터**
