# 서비스 도메인 정의

> 작성: 2026-07-09 (시니어 PM 관점 정리)
> 목적: "이 서비스가 무엇이고 무엇이 아닌지"를 명확히 하여, 기능 제안·우선순위 판단·리뉴얼 방향의 기준선으로 삼는다.

---

## 1. 도메인 한 줄 정의

**"부동산 의사결정을 위한 시군구 단위 실거주인구 통계 리서치 도구"**

행정안전부 주민등록 인구통계(공공 데이터)를 가공하여, 특정 지역의 인구 흐름을 부동산 구매·이주 판단의 참고 지표로 제공하는 **정적 데이터 시각화 서비스**다. 실시간 API, 예측 모델, 개인 식별 데이터를 다루지 않는다 — 월 단위로 갱신되는 공개 집계 통계를 분석·비교·시각화하는 것이 도메인의 전부다.

---

## 2. 상위 도메인: 부동산 의사결정 지원

- **핵심 가정:** 인구 증감은 주거 수요·상권 활성도·지역 개발 가능성의 선행/동행 지표다 (`/about` 페이지의 서비스 소개 문구가 이 가정을 명시적으로 서술).
- **인접하지만 다른 도메인과의 경계:**
  - 부동산 실거래가·시세 정보 ❌ (다루지 않음 — 국토부 실거래가 API 등과는 무관)
  - 인구 예측/추계 모델 ⚠️ (상세 페이지에 "향후 6개월 예측 오버레이"가 있으나 통계청 수준의 공식 추계가 아닌 단순 추세 연장이며, 투자 판단의 근거로 오인되지 않도록 면책 조항(`/privacy`, `/about`)으로 제한)
  - 상권/생활 인프라 분석 ❌ (다루지 않음)
  - 인구 통계 일반(전국/광역 단위 거시 지표) △ (전국 총괄 현황은 홈 화면에 보조적으로만 노출, 핵심은 시군구 단위)

## 3. 하위 도메인 (기능 축)

현재 코드베이스는 아래 4개 하위 도메인으로 나뉘며, 이는 `BottomNav`의 4탭(홈/지도/순위/비교)과 대체로 일치한다.

| 하위 도메인 | 질문에 답한다 | 핵심 페이지 |
|---|---|---|
| **지역 프로파일링** | "이 동네 인구가 어떻게 변해왔는가?" | `/[sido]/[sigungu]`, `/[sido]/[sigungu]/detail` |
| **전국 랭킹/탐색** | "어디가 뜨고 있고 어디가 빠지고 있는가?" | `/ranking`, `/trending`, `/map` |
| **상대 비교** | "A 지역과 B 지역 중 어디가 나은가?" | `/compare` |
| **개인화 탐색** | "내가 관심있는 곳만 빠르게 다시 보고 싶다" | 홈의 즐겨찾기/최근 본 지역 (localStorage, 서버 저장 없음) |

## 4. 데이터 도메인 경계

- **단위:** 시군구(10자리 MOIS 코드) — 그 이하(읍면동)나 그 이상(광역시도 단독 분석)은 부차적으로만 지원.
- **지표 종류:** 총인구·세대수·세대당인구·성비·연령 구조(10세 단위)·순이동 근사치. 소득·주택가격·산업 등 타 통계는 도메인 밖.
- **시간 해상도:** 월 단위. 일/주 단위 변화나 실시간성은 다루지 않는다(주민등록 통계 자체가 월 단위 공표이므로 구조적 한계).
- **갱신 방식:** 수동 — 담당자가 `data/raw/`에 CSV를 추가하고 `npm run build-data`를 실행해야 반영됨. 자동 크롤링/API 연동 없음(계획된 백로그 M-4).

## 5. 사용자 정의

| 페르소나 | 니즈 | 주로 쓰는 하위 도메인 |
|---|---|---|
| 내 집 마련 실수요자 | 이사/매수 후보 지역의 인구 추세·연령 구조 확인 | 지역 프로파일링, 비교 |
| 갭투자/임장 리서처 | 여러 지역을 빠르게 스캔해 유망 지역 스크리닝 | 랭킹, 트렌딩, 지도 |
| 이주 검토자(비부동산 목적) | 특정 지역의 인구 감소·고령화 여부 확인 | 지역 프로파일링 |

## 6. 도메인 밖 (Out of scope, 현재 기준)

- 회원가입/로그인, 서버 사이드 개인화 저장 (알림·구독은 M-1 백로그로 검토 중이나 미착수)
- 실시간 데이터, 외부 API 연동
- AI 기반 해설/인사이트 생성 (M-2 백로그)
- 유료 리포트/구독 모델 (M-3 백로그)
- 데스크톱 최적화 레이아웃 (의도적으로 모바일 전용 유지하기로 결정됨)

---

**정리:** 이 서비스는 "인구 통계 서비스"라기보다 **"부동산 리서치용 인구 데이터 렌즈"**로 정의하는 것이 정확하다. 새 기능을 검토할 때는 "이것이 시군구 단위 실거주인구 데이터를 통해 부동산/이주 의사결정에 도움을 주는가?"를 1차 필터로 삼는다.
