## 2. 사용자 여정 분석

### 페르소나 A: 부동산 실수요자 (아파트 매수 검토 중)
- **목적**: "이 동네 인구 느는지, 젊은 사람 많은지 확인"
- **주요 경로**: 검색 → 지역 요약 → 연령 탭 → 비교
- **핵심 니즈**: 지역별 연령대 비교, 전입전출 동향, 직관적 요약
- **현재 불편**: 전입전출 데이터 없음, 비교 UX 번거로움

### 페르소나 B: 부동산 투자자 (임장 전 데이터 리서치)
- **목적**: "인구 증가 지역 발굴, 순위 비교"
- **주요 경로**: 트렌딩 → 순위 → 지역 비교
- **핵심 니즈**: 빠른 상위/하위 지역 탐색, 리포트 저장·공유
- **현재 불편**: 지도 탐색 불가, 데이터 내보내기 없음

### 페르소나 C: 이주 준비 사용자 (직장/학교 이전)
- **목적**: "새 동네 인구 구성, 세대 규모 파악"
- **주요 경로**: 검색 → 지역 요약 → 상세 → 공유
- **핵심 니즈**: 공유 편의성, 연령 구조 한눈에 보기
- **현재 불편**: 카카오톡 공유 시 OG 이미지 없어 미리보기 없음

## 3. 실행 로드맵

### 즉시 (0~1주)

| # | 기능 | 설명 | 파일 |
|---|------|------|------|
| I-1 | B1: 비교 UX 개선 | 비교 버튼 클릭 시 기존 region_a 유지 후 region_b 자동 채우기 | `src/app/[sido]/[sigungu]/page.tsx` |
| I-2 | B2: 광고 조건 완화 | `region_a && region_b` → `region_a \|\| region_b` | `src/app/compare/page.tsx` |
| I-3 | Ranking URL 상태 반영 | 정렬·필터를 URL에 반영하여 공유·북마크 가능 | `src/components/ranking/RankingClient.tsx` |

### 1순위 — SEO 기반 유기적 유입 (1~2주)

| # | 기능 | 설명 | 파일 |
|---|------|------|------|
| S-1 | 동적 메타태그 | 각 지역 페이지에 title, description 주입 | `src/app/[sido]/[sigungu]/page.tsx` |
| S-2 | OG 이미지 자동 생성 | next/og로 지역별 동적 소셜 카드 이미지 | `src/app/og/route.tsx` |
| S-3 | 사이트맵 자동 생성 | 269개 지역 경로 포함 sitemap.xml | `src/app/sitemap.ts` |
| S-4 | robots.txt | 크롤러 접근 허용 설정 | `src/app/robots.ts` |
| S-5 | 지역 인사이트 요약 | 각 지역 페이지에 자연어 통계 요약 텍스트 | 지역 요약 페이지 하단 |

### 2순위 — 데이터 깊이 확장 (2~4주)

| # | 기능 | 설명 |
|---|------|------|
| D-1 | 전입전출 탭 추가 | 지역 상세 탭에 전입·전출 월별 차트 및 인사이트 카드 |
| D-2 | 인구 예측 시각화 | 최근 12개월 추세 기반 향후 6개월 예측선(점선) 오버레이 |
| D-3 | 홈 인기지역 탭화 | "급증 / 급감 / 고령화" 탭 전환으로 더 많은 진입점 제공 |

### 3순위 — 탐색성 강화 (2~3개월)

| # | 기능 | 설명 |
|---|------|------|
| E-1 | 지도 기반 탐색 | SVG/GeoJSON 히트맵으로 시도별 시각적 탐색 (react-simple-maps) |
| E-2 | 비교 기능 고도화 | 최대 3개 지역 비교, 연령 구조 비교 탭 추가 |
| E-3 | 카카오 공유 SDK | 썸네일 + 버튼 텍스트 커스터마이즈 |
| E-4 | PC 레이아웃 완성 | 사이드바 + 넓은 차트 레이아웃 |

### 4순위 — 수익화 & 리텐션 (3개월+)

| # | 기능 | 설명 |
|---|------|------|
| M-1 | 알림/구독 | 관심 지역 월별 통계 이메일 요약 (Resend + Vercel Cron) |
| M-2 | AI 인사이트 | Claude API 기반 지역별 자연어 인사이트 (SSG 빌드 타임 생성) |

## 5. 서비스 방향성

### 서비스 오너 관점

- **단기**: "검색에서 찾히는 서비스"로 만들기 — SEO + AdSense 실제 적용
- **중기**: "부동산 데이터 리서치의 기본 도구"로 포지셔닝 — 전입전출 + 예측 + 지도
- **장기**: 수익 모델 다각화 — AdSense → 프리미엄 리포트 → B2B API

### 사용자 관점

사용자가 원하는 것:
1. "내가 보려는 지역 인구가 느는지 주는지 빨리 알고 싶다" → 현재 충족
2. "이 지역에 실제로 사람이 들어오는지 나가는지" → **전입전출 데이터 부재**
3. "다른 지역이랑 쉽게 비교하고 싶다" → 비교 UX 개선 필요
4. "친구/배우자에게 공유하고 싶다" → OG 이미지 + 카카오 공유 필요

**최종 목표**: 단순 통계 열람 서비스 → **부동산 의사결정 지원 플랫폼**

## 6. 핵심 요약

1. **지금 당장**: SEO(메타태그+OG이미지+사이트맵)와 백로그 B1/B2 해결 — 유입과 UX의 기반을 먼저 깔아야 다음 기능이 의미 있다.
2. **2개월 안에**: 전입전출 데이터 추가 + 비교 UX 개선 — 부동산 구매자 페르소나가 진짜 원하는 데이터를 채워 재방문율을 높인다.
3. **장기 방향**: 데이터 깊이와 공유 편의성으로 경쟁 서비스와 차별화, 부동산 의사결정 지원 플랫폼으로 진화.
