## 2. 사용자 여정 분석

### 페르소나 A: 부동산 실수요자 (아파트 매수 검토 중)
- **목적**: "이 동네 인구 느는지, 젊은 사람 많은지 확인"
- **주요 경로**: 검색 → 지역 요약 → 연령 탭 → 비교
- **핵심 니즈**: 지역별 연령대 비교, 전입전출 동향, 직관적 요약
- **현재 불편**: 전입전출 데이터 없음, 비교 UX 번거로움

### 페르소나 B: 부동산 투자자 (임장 전 데이터 리서치)
- **목적**: "인구 증가 지역 발굴, 순위 비교"
- **주요 경로**: 트렌딩 → 순위 → 지역 비교
- **핵심 니즈**: 빠른 상위/하위 지역 탐색, 리포트 저장·공유
- **현재 불편**: 지도 탐색 불가, 데이터 내보내기 없음

### 페르소나 C: 이주 준비 사용자 (직장/학교 이전)
- **목적**: "새 동네 인구 구성, 세대 규모 파악"
- **주요 경로**: 검색 → 지역 요약 → 상세 → 공유
- **핵심 니즈**: 공유 편의성, 연령 구조 한눈에 보기
- **현재 불편**: 카카오톡 공유 시 OG 이미지 없어 미리보기 없음

## 5. 서비스 방향성

### 서비스 오너 관점

- **단기**: "검색에서 찾히는 서비스"로 만들기 — SEO + AdSense 실제 적용
- **중기**: "부동산 데이터 리서치의 기본 도구"로 포지셔닝 — 전입전출 + 예측 + 지도
- **장기**: 수익 모델 다각화 — AdSense → 프리미엄 리포트 → B2B API

### 사용자 관점

사용자가 원하는 것:
1. "내가 보려는 지역 인구가 느는지 주는지 빨리 알고 싶다" → ✅ 충족
2. "이 지역에 실제로 사람이 들어오는지 나가는지" → ✅ 전입출 탭(순이동 근사)으로 부분 충족
3. "다른 지역이랑 쉽게 비교하고 싶다" → ✅ 비교 UX + 연령 구조 비교 완료
4. "친구/배우자에게 공유하고 싶다" → ✅ OG 이미지 + URL 공유 완료
5. "지도에서 어느 지역이 핫한지 보고 싶다" → **E-1 지도 탐색으로 해결 예정**

**최종 목표**: 단순 통계 열람 서비스 → **부동산 의사결정 지원 플랫폼**
